# 단계 0 하드웨어 기준선 체크리스트

## 목적

STM32와 ROS 코드를 작성하기 전에 두 SO-ARM101의 실제 서보 구성과 전원 상태를 측정한다.

## 안전 준비

- [ ] 한 번에 한 팔만 전원 연결
- [ ] 팔 주변에 사람과 물체가 없음
- [ ] 낮은 속도와 가속도 사용
- [ ] 전원 스위치 또는 플러그에 즉시 접근 가능
- [ ] 기구가 예상치 못하게 움직여도 충돌하지 않는 자세
- [ ] 멀티미터 측정 범위와 극성 확인

`최소 raw`와 `최대 raw`는 기구적 끝까지 강제로 움직여 찾지 않는다. 충돌과 케이블 장력을 고려한 보수적인 software limit을 기록한다.

## C. 상태값(Feedback) 확인

| Bus | ID | position | speed | load | voltage | 응답 안정성 | 비고 |
|---|---:|---:|---:|---:|---:|---|---|
| left | 1~6 |  |  |  |  |  |  |
| right | 1~6 |  |  |  |  |  |  |

확인 항목:

- [ ] 서보 12개의 ping 성공
- [ ] position 값이 실제 움직임과 같은 방향으로 변화
- [ ] 정지하면 speed가 정상적으로 감소
- [ ] voltage 값이 공급 전압과 크게 다르지 않음
- [ ] 반복해서 읽을 때 packet 오류가 누적되지 않음

## D. 재부팅과 기본 상태

- [ ] Torque Disable 상태에서 전원을 넣었을 때 명령하지 않은 움직임 없음
- [ ] ID와 baud rate가 유지됨
- [ ] 현재 위치를 읽을 수 있음
- [ ] 잘못된 ID에 응답하지 않음
- [ ] USB adapter 재연결 후 정상 복구

## E. 단계 0 완료 조건

- [ ] `HW-001`부터 `HW-005`까지 결과 기록
- [ ] 12축의 ID와 관절 연결 관계 확정
- [ ] 보수적인 raw software limit 기록
- [ ] 전원 전압 측정 완료
- [ ] 비정상 서보와 예비 교체 필요 여부 판단
- [ ] 결과를 `docs/PORTFOLIO_LOG.md`에 요약

단계 0을 통과하기 전에는 STM32에서 전체 팔 이동 명령을 실행하지 않는다.

## F. 기계 판정 파일

측정값은 `hardware/phase0_baseline.json`에도 입력한다. 오른팔만 먼저 측정한 경우 다음 명령으로 오른팔 게이트만 판정한다.

```bash
python3 tools/validate_phase0.py --arm right
```

양팔 측정 완료 후 전체 게이트를 판정한다.

```bash
python3 tools/validate_phase0.py --arm all
```

파일에 `null`이 남아 있거나, 관절 이름이 중복되거나, raw 현재 위치가 기록된 안전 범위를 벗어나거나, fault 항목이 안전 상태와 모순되면 검증기는 실패해야 한다. 측정 전 실패는 정상이며 임의의 값으로 통과시키지 않는다.
